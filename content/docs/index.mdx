CCPEDIA is a knowledge layer for Canton developers. We sync every official Canton surface (forum, GitHub, docs, CIPs, mailing lists, videos, dev fund, ecosystem registry, validator calendar, app metrics) and expose all of it through one MCP endpoint.

Your AI tool connects once and gets 78 tools that know about Canton.

## Why this exists

Canton reference material lives in eight separate places. The forum, the docs site, the CIPs repo, four mailing lists, GitHub issues across a dozen repos, the Foundation knowledge base, the dev fund tracker, the SV calendar, the ecosystem registry, plus a few hundred hours of YouTube talks. AI coding assistants only see what gets pasted into the prompt.

CCPEDIA syncs all of it continuously and serves it through one MCP endpoint at [`ccpedia.xyz/mcp`](https://ccpedia.xyz/mcp).

## Layers

CCPEDIA exposes the same Canton data through several interfaces. Pick the one that fits your client.

| Layer | What it is | Best for |
| --- | --- | --- |
| MCP server at `/mcp` | 78 tools your AI calls directly | Claude, Cursor, Windsurf, Zed |
| REST API v1 at `/api/v1/*` | Same data as HTTP | Dashboards, custom apps |
| Atom feeds at `/feed/*` | Per-source firehose | RSS readers, monitoring |
| Web UI at `ccpedia.xyz` | Human search and browse | Developers, governance |
| docs.ccpedia.xyz | Tool reference, guides, patterns | AI tools and humans wiring up |
