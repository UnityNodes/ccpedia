CCPEDIA serves its MCP at **`https://ccpedia.xyz/mcp`**. No auth. Rate limit is 60 calls per 15 minutes per IP. The same URL works for every supported AI client.

## Step 2. Verify the connection

After adding CCPEDIA, your client should see **78 tools**. Sanity-check from the terminal:

```bash
curl -s -X POST https://ccpedia.xyz/mcp \
  -H 'content-type: application/json' \
  -H 'accept: application/json, text/event-stream' \
  -d '{"jsonrpc":"2.0","id":1,"method":"tools/list","params":{}}'
```

You should see roughly 78 `tools` entries in the response.

## Step 3. Ask a useful question

Pick any of these. Just ask in plain language. Tool routing happens automatically.

<Accordions type="single">
  <Accordion title="What's the current Canton SDK version?">
    Routes to `get_current_versions`. Joins github_releases tags with validator-announce mailing posts.
  </Accordion>
  <Accordion title="Diagnose this error: TOPOLOGY_TOO_MANY_PENDING_TOPOLOGY_TRANSACTIONS">
    Routes to `diagnose_error`. Tokenises the error and searches forum threads and GitHub issues for prior fixes.
  </Accordion>
  <Accordion title="Show dev fund proposals related to validator monitoring">
    Routes to `search_community` and `detect_builder_overlap`. Surfaces existing work to avoid duplication.
  </Accordion>
  <Accordion title="What does CIP-0056 require, and is it ready?">
    Routes to `get_cip_implementation_status` and `detect_cip_dependency_chain`. Full readiness graph.
  </Accordion>
  <Accordion title="When is the next MainNet upgrade?">
    Routes to `get_upcoming_operations`. Pulled from the SV calendar every 12 hours.
  </Accordion>
</Accordions>

## Step 4. Pin the URL
